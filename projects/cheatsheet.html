SQL Cheat Sheet

Naming Conventions

  • snake_case
  • Plural Table Names
  • Auto-increment id field FIRST
  • Date Fields generally end in _at or _on, except when field is obviously something else
  • If not automatically added (rails), add created_at and updated_at fields

Terminal Commands

  • Open shell by sqlite3 filename.sqlite3
  • .quit
  • .schema

SQL SELECT Statements

  • Standard: SELECT column_name or * FROM table_name WHERE condition AND/OR Condition ORDER BY column_name [ASC|DESC]
  • Between: SELECT column_name(s) or * FROM table_name WHERE column_name BETWEEN value1 AND value2
  • Inner Join: SELECT column_name(s) FROM table_name1 INNER JOIN table_name2 ON table_name1.column_name=table_name2.column_name

    • Operators in Where

    Operator Description
    = Equal
    <> or != Not equal.
    > Greater than
    < Less than
    >= Greater than or equal
    <= Less than or equal
    BETWEEN Between an inclusive range
    LIKE Search for a pattern wildcard %
    IN To specify multiple possible values for a column

    Relationships

    • One-to-Many Relationships - the most common kind, where one record has many children records
    • One-to-One Relationships - for optional or additional details for one record
    • Many-to-Many Relationships - Joining two tables together where each is in a one to many relationship with a join table

    Joins

    • Inner Join- Returns all rows when there is at least one match in BOTH tables
    • Left Join - Return all rows from the left table, and the matched rows from the right table
    • Right Join - Return all rows from the right table, and the matched rows from the left table
    • Full Join - Return all rows when there is a match in ONE of the tables

    SQL INSERT/UPDATE/DELETE

    • INSERT INTO table_name (column1,column2,column3,...)VALUES (value1,value2,value3,...);
    • UPDATE table_name SET column1=value1,column2=value2,... WHERE some_column=some_value;
    • DELETE FROM table_name WHERE some_column=some_value;

    SQL UNION PROUD

    • ONLY Distinct ValuesSELECT column_name(s) FROM table1 UNION SELECT column_name(s) FROM table2;
    • Allow Duplicate values SELECT column_name(s) FROM table1 UNION ALL SELECT column_name(s) FROM table2;
    • Can also be used WHERE statements, joins, etc.
