Arrays and Hashes

A satisfactory explanation

Arrays and hashes are very similar ways of organizing data in a way that it can be accessed easily through a number of methods. Arrays are like an list. It's one object or line after another. It can be ordered in different ways. Here's an example:

["Johnny", "Dee Dee", "Joey", "Tommy"]

Hashes are harder to explain, as I found when trying to explain them to my girlfriend. Part of it is the name- I'm sure there's some good technical reason for "hash", but it sounds like a bunch of things mixed up together, like a breakfast hash. I prefer the name other languages use, "dictionary." That brings up an obvious metaphor, of entries in a dictionary, with a word, and then a definition.

That isn't far from the actual use. Instead of the indexes that arrays use, hashs have keys, which must be unique. To rewrite the above example:

["guitarist" => "Johnny","bassist" => "Dee Dee","vocalist" =>"Joey", "drummer"=> "Tommy"]

Both arrays and hashes mix-in the Enumerable module. "Mix-in"s are a powerful way of sharing code between classes. For any class you can simply type:

include Enumerable

and you can use all the methods of Enumerable. Many of the most used methods of arrays and hashes are actually part of the enumerable module. For example, you can sum a (numerical) array by using the reduce method.

A.reduce(:+)

Whereas to sum the values of a hash, you use

H.values.reduce(:+)

In either case, the method is available thanks to Enumerables. Reduce is a more complicated function than this example indicates(it would return an error if the array contained strings), but it can do powerful things that make up for the fact that there's no sum or factorial.

When would you use hashes? When would you use arrays? These are good questions, and what it comes to is, how are you organizing your data. If you just want to maintain an ordered list of values, arrays are the way to go. If you want the values associates with some other value- like a key, values are the way to go. With that you can have a more complex structure. Key value pairs are easier to transfer to XML. Of course you can also make arrays of hashes. Or hashes of arrays.
